Detailed Description

Allows programs to log relevant information.

This class allows the user to define any number of message types, each of which can be configured to output to the console, to a log file, both or neither, using the enum specified in output.h. Each of these message types can also be logged to the console in different colors.

This class comes with 6 predefined types:

  • general: general output format. when in doubt, use this type
  • warning: if something strange has happened that can be recovered from, such as receiving a null parameter, use this type to complain about it
  • error: use this if something has gone seriously wrong. should never be disabled. exceptions should complain by logging an error
  • success: if some test or check was passed, this would be the format to use
  • state: useful for outputting the value of variables for debugging purposes
  • location: useful for outputting the current location of program flow for debugging purposes

This class is used to log information by calling its overloaded operator()().

ex:

Log log("a log", toConsole);

// first use of operator()() log("this is a general log message"); // second use log("warning", "this is a warning about something");

Every class should have a private Log object and use it to optionally output as much information as possible, but turn of the unimportant parts by default.

Messages are logged to a newly created file, whose name takes the format name_MMDDYYYY_HHMMSS.log, with name being the name passed to the constructor, except converted to all_lower_snake_case, followed by the date then time.

New log formats can be created by passing a LogType object to the registerType() function.

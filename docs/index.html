BEN the Robot Documentation

these are a few C++ classes that I find useful for squashing bugs.

Their documentation can be viewed at http://gabrield42.github.io/debug/.

Log

Log is useful for outputting information to both a log file as well as to the terminal. It is mainly used by calling its overloaded operator()(). It is extremely configurable because not only does it defines a handful of different message types and allows where they output to be set individually, it allows the user to introduce additional types. Each type's background and text color can be set as well.

Message Types

At the moment, there are 6 default output types:

  • general: general output format. when in doubt, use this type
  • warning: if something strange has happened that can be recovered from, such as receiving a null parameter, use this type to complain about it
  • error: use this if something has gone seriously wrong. should never be disabled. exceptions should complain by logging an error
  • success: if some test or check was passed, this would be the format to use
  • state: useful for outputting the value of variables for debugging purposes
  • location: useful for outputting the current location of program flow for debugging purposes

Usage

Here is a simple usage example:

Log log("General Log 1", Output::toConsole); log("this is a general message"); log("warning", "this is a warning message"); log("error", "this is a error message"); log("success", "this is a success message"); log("state", "this is a state message"); log("location", "this is a location message");

That will write

this is a general message this is a error message this is a success message

to the console and

warning: this is a warning message error: this is a error message

to the file general_log_1_MMDDYYYY_HHMMSS.txt, where MMDDYYYY corresponds to month-day-year, and HHMMSS is the current time.

Note that if the message type is omitted, it is assumed to be a general message.

Register Custom Log Types

To register a new type, use the registerType(const LogType newType, const Output::type toWhere = Output::toNowhere) function. The output format is defaulted in Log rather than LogType because it can be set on a per object basis, rather than globally.

Here is a simple example of registering a new type and using it:

Log log("test log", Output::toConsole); LogType testType("my log type", Color::FG_BLUE, Color::BG_LIGHT_GREY); log.registerType(testType, Output::toConsole); log("my log type", "this is a message of custom user type 'my log type'");

Test

This is a simple class for performing unit tests. It provides 4 types of assertions, and logs their success using the Log class. upon object destruction, it reports how the test as a whole fared.

Usage

Here is a simple example that uses all 4 types of assertions:

Test test("a simple test"); test.assert("simple addition", 1 + 1 == 2); // returns true test.assertFalse("more addition", 1 + 2 == 3); // returns false test.assertNull("null", NULL); // returns true test.assertNotNull("object pointer", &test); // returns true

running this program will create this logfile:

general: starting test a simple test success: test simple addition passed error: test more addition failed success: test null passed success: test object pointer passed error: test suite a simple test failed. Examine logfile for details.
